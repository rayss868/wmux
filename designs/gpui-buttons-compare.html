버튼 리디자인 — 현재 vs gpui-style

"gpui-style"는 Rust 프레임워크가 아니라 Zed 앱의 룩앤필이에요 — 전부 CSS로 재현됩니다. DESIGN.md가 이미 Zed · quiet chrome 계보를 명시했으니, 이건 노선 변경이 아니라 플랫을 "미완성"이 아닌 "의도된 미니멀"로 마감하는 것. 핵심 재료는 그림자·글로우가 아니라 미세한 surface lift + 상단 1px inset 하이라이트(이미 sidebar-row-active가 쓰는 트릭). 아래 버튼에 hover·click 해보세요.

구현은 "레시피 1개"면 대부분 풀려요

전부 새 CSS가 아니라, 산재한 ad-hoc Tailwind 버튼을 공유 <Button variant> + 토큰으로 모으는 일입니다. 핵심은 secondary의 이 4줄 — DESIGN.md의 "elevation 3레벨" 안에 있고(글로우 아님), color-mix라 8개 테마 전부 자동 대응돼요. 

/* gpui-style secondary — 나머지 variant는 이걸 변주 */
background: color-mix(in srgb, var(--bg-surface) 72%, transparent);
border: 1px solid rgba(255,255,255,.09);
box-shadow: inset 0 1px 0 rgba(255,255,255,.045);   /* ← 상단 하이라이트 = "깎아만든" 느낌 */
border-radius: 5px;                                   /* DESIGN.md 4px → 5px 소폭 완화 제안 */
바뀌는 것 · box-shadow inset 하이라이트, surface fill, 반경 4→5px, active 눌림 안 바뀌는 것 · amber 다이어트(5±2), 그라디언트 금지, hairline, terminal 히어로

DESIGN.md 변경 필요 항목: 버튼 반경 4px → 5px, Component Rules에 "buttons: surface-lift + top inset-highlight" 1줄 추가. 오너 승인 대상.
